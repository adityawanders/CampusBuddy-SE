# MoSCoW Prioritization – CampusBuddy

## Must Have (Critical Features)
These are essential for the system to work.

1. Student Login
2. Student Registration
3. Room Details Setup (Hostel block + room number)
4. Raise Complaint (category + description)
5. Complaint Category Selection
6. Track Complaint Status (Pending → Assigned → In Progress → Completed)
7. Warden Dashboard (basic overview)
8. View All Complaints (Warden)
9. Assign Complaints to Staff
10. View Assigned Tasks (Maintenance staff)

---

## Should Have (Important but not mandatory)
These improve usability and control.

1. Complaint Photo Upload
2. Set Complaint Priority (Low/Medium/High)
3. Warden Status Update
4. Update Task Progress (Staff)
5. Complaint Update Notifications

---

## Could Have (Nice-to-have / Optional features)
These improve experience, but can be implemented later.

1. Complaint History
2. Mark Complaint as Resolved
3. Reopen Complaint
4. Mess Feedback Submission
5. Hostel Announcements

---

## Won’t Have (Not planned for current version)
These are out of scope for this assignment prototype.

1. AI-based complaint auto-classification
2. Live chat support
3. Payment integration
4. Smart IoT-based automatic issue detection
5. Multi-hostel multi-campus scaling system
